# CLAUDE.md ‚Äî Elysium Docs

## Purpose
Public documentation site for ELYSIUM, a no-code AR worldbuilding platform by MYTHOLOGI Inc. Built with Docusaurus and deployed at [docs.elysium.ar](https://docs.elysium.ar). Documents features across the whole platform (Creation App, Portal, GEIST Engine, Wallet, Analytics, Reality Bridge, and the broader ecosystem) ‚Äî it has no runtime code that other repos depend on.

## Platform Context
- **Depends on:** nothing at runtime ‚Äî standalone static site.
- **Consumed by:** end users / creators (public site); no repo imports it.
- **Documents:** the full ELYSIUM platform (features sourced from `elysium-app`, `elysium-web-app`, and the broader ecosystem).
- **Design language:** follows the ELYSIUM dark glassmorphism aesthetic (shared cross-platform design hierarchy; implemented here in `src/css/custom.css`, not a copy of any other repo's tokens).

## Tooling & Commands
```bash
npm start              # Dev server + hot reload, port 3000 (shows ALL content incl. drafts/unpublished)
npm run build          # Production build (excludes drafts + unpublished pillars)
npm run serve          # Serve the production build locally
npm run typecheck      # tsc type-check
npm run dino           # Launch Docu Dino docs manager ‚Üí http://localhost:3333
npx docusaurus clear   # Clear Docusaurus cache (use when style/config changes don't reflect)
```
- **No build step for Tailwind** ‚Äî it's wired via an inline PostCSS plugin inside `docusaurus.config.js` (`configurePostCss` pushes `tailwindcss` + `autoprefixer`), not a standalone config entry.
- `dotenv` is required at the top of `docusaurus.config.js`, but **no `process.env.*` is actually consumed** today ‚Äî `url` is hardcoded to `https://docs.elysium.ar`. There is **no `.env.example`** in the repo.
- Requires `.npmrc` (`@mythologi-xr:registry=https://npm.pkg.github.com`) for the GitHub Packages registry.

## Rules (Enforced)
- `[CI]` `onBrokenLinks: 'throw'` in `docusaurus.config.js` ‚Äî a broken internal link **fails the build**; broken anchors/markdown links/images are `'warn'`.
- `[convention]` Use **npm, never yarn** ‚Äî the `README.md` still shows `yarn` commands and is stale; ignore it.
- `[convention]` Add docs by creating folders/files under `docs/` ‚Äî the sidebar is **autogenerated** from folder structure + `_category_.json` files (`sidebars.js` uses `{ type: 'autogenerated' }` per pillar). Do not hand-author a flat sidebar tree.
- `[convention]` Gate content visibility through `docs/publishing/docFlags.js` (pillar booleans + optional page-level overrides), not by deleting files. Both `sidebars.js` and `docusaurus.config.js` navbar read `pillars` from it.
- `[advisory]` Edit layout chrome only in the swizzled components under `src/theme/` (`Navbar/`, `Footer/`).

## Hard Constraints & Anti-Patterns
- **Do not enable a light-mode toggle** ‚Äî dark mode is the **default and only** mode (`colorMode: { defaultMode: 'dark', disableSwitch: true }`). All Infima + glassmorphism overrides live under `[data-theme='dark']`; there is no maintained light theme.
- **Do not re-enable Tailwind Preflight** ‚Äî it is disabled (`corePlugins: { preflight: false }` in `tailwind.config.js`) to avoid clobbering Infima's base styles. Re-enabling it breaks Docusaurus typography.
- **Do not hardcode border radii** ‚Äî derive from `--global-radius: 12px` (single source of truth; feeds `--ifm-global-radius`, `--glass-radius`, and the Tailwind `rounded-*` scale).
- **Do not assume the homepage is a doc** ‚Äî `src/pages/index.tsx` is a standalone page outside the docs tree (docs are served at site root via `routeBasePath: '/'`).

## Gotchas & Tribal Knowledge
- **Dark mode is the only mode** ‚Äî there is no light theme to test against; `disableSwitch: true` hides the toggle entirely.
- **npm, not yarn** ‚Äî the committed `README.md` is the original Docusaurus scaffold and still says `yarn ...`; it is wrong. Use the `package.json` scripts.
- **Sidebar is auto-generated from `docs/` folder structure** via `_category_.json` files and `autogenerated` items ‚Äî there is no manual ordering list to keep in sync; reorder/reorganize through folders or Docu Dino.
- **Docu Dino** (`npm run dino`, port 3333) is an Express + vanilla-JS tool (no build step) that reads/writes `docs/` frontmatter and `_category_.json` directly, with real-time **Server-Sent Events** file-watcher sync. It drag-and-drops page/category reorder, edits titles/content inline, toggles page `draft` frontmatter, toggles pillar flags in `docs/publishing/docFlags.js`, previews markdown, and supports 12-locale i18n.
- **Draft + unpublished content is dev-only** ‚Äî `draft: true` frontmatter and `false` pillar flags are visible under `npm start` but excluded from `npm run build`. Verify visibility decisions against a production build, not the dev server.
- **Glassmorphism + Infima + Tailwind coexist** ‚Äî Tailwind's `colors.gray` maps onto Infima CSS variables for dark-mode compatibility, and Preflight is off so the three don't fight. Apply the reusable `.glass` class (`src/css/custom.css`) for the blur+transparent-border effect.
- **Fonts:** Manrope (`--ifm-font-family-base`, Google Fonts) is the primary; **Monument Extended** (`PPMonumentExtended-Regular.*`, loaded locally from `src/fonts/MonumentExtended/`) is the display font for homepage headings; Noto Sans Mono (Google Fonts) is the monospace homepage copy. A missing local Monument Extended file silently falls back to system fonts.
- **i18n is 12-declared / 3-populated** ‚Äî `docusaurus.config.js` declares 12 locales (en, ar, de, es, fa, fr, ja, ko, pt, pt-BR, ru, zh; `ar`/`fa` are RTL), but only `i18n/fr`, `i18n/ja`, `i18n/zh` have translation files on disk. Don't assume all 12 are translated.
- **`@docusaurus/plugin-client-redirects` is installed but NOT wired** ‚Äî it's in `package.json`/`node_modules` but absent from the `plugins` array, so no redirects are active; docs are served at root (`routeBasePath: '/'`), not under `/docs`.
- **No CI / context-node lint vendored yet** ‚Äî there is no `.github/workflows/` and `.claude/` holds only `settings.local.json`. The footer marker below opts this root in once the context-node scripts land.
